Layer Descriptions

Presentation Layer: User-facing dashboard with real-time monitoring views

Application Layer: Core backend services handling security and context aggregation

Intelligence Layer: ML models and AI assistant for threat detection

Data Layer: Time-series metrics, logs, and process monitoring

Infrastructure Layer: System monitoring agents and critical services

Alerting System: Multi-channel notification delivery

Audit & Integrity Layer: Blockchain-based immutable audit trail

Phase Summary

Phase 1 (Weeks 1-2): Foundation & Infrastructure - Feb 1-14

Phase 2 (Weeks 3-4): ML Models & Core Features - Feb 15-28

Phase 3 (Week 5): Intelligence & User Experience - Mar 1-7

Phase 4 (Week 6): Integration, Testing & Demo - Mar 8-15

Team Tracks

Track A (3 members): Backend, ML Models, Infrastructure, Security, Blockchain

Track B (2 members): Frontend, UI/UX, Dashboard, Demo

Technology Stack Highlights

Frontend/UI: Laravel Blade + Tailwind (Server-side rendered, responsive dashboard)

Orchestrator: Laravel (PHP 8.x) - Core logic: Auth, RBAC, Eloquent DB, Alert Queues

ML Engine: FastAPI (Python) - High-performance API for AI models (Isolation Forest, NLP)

Auth System: Laravel Sanctum - Secure session and token-based authentication

Alerting: Laravel Queues - Asynchronous multi-channel notifications (Telegram/SMS)

Metrics: Prometheus - Time-series database with PromQL

Logs: Loki + Promtail - Log aggregation for ML analysis

Audit & Integrity: Hyperledger Fabric (Blockchain) - Immutable, tamper-evident ledger for server state changes, security events, and administrative actions

Blockchain Integration Details

Purpose: Provides an immutable audit trail ensuring transparency and accountability

What Gets Recorded:

  • Server state changes and configuration modifications
  • Security events and threat detections
  • Administrative actions and access logs
  • Alert generation and response actions

Benefits:

  • Tamper-proof audit logs for compliance and forensics
  • Cryptographic verification of all security events
  • Transparent accountability for system changes
  • Enhanced trust in security monitoring data
